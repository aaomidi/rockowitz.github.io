Preliminary Notes for ddcutil Release 1.1.1

The changes in release 1.1.1 largely affect only shared library libddcutil. Users of the command line program ddcutil do not need to update.

Logging

Two forms of logging now exist for the shared library. These were implemented to address situations where there is no terminal to write to, e.g. PowerDevil in KDE Plasma.

libddcutil trace file

ddcutil can redirect all output that it would normally write to the terminal to a trace file instead. The file is specified using option --libddcutil-trace-file. This option can be only be specified in the [libddcutil] section of the ddcutil configuration file, ddcutilrc.

The file name given can be either absolute or relative. If relative, the directory is resolved as per the definition of XDG_DATA_HOME in the XDG specification. Typically this is $HOME/.local/share/ddcutil.

Syslog

Major events are written to the system log. These include starting, stopping, and severe internal errors.

API changes

New Function Comments
ddca_add_trace_groups  Add to the set of traced groups
ddca_get_extended_version_string  Version string with optional suffix, e.g. "1.1.2-rc2"

Note: By comparison, ddca_set_trace_groups() replaces the existing trace group definition.
New Macro Comments
DDCUTIL_VSUFFIX  version suffix if it exists

### Miscellaneous

  • Additional checks are performed on DDCA_Display_Ref function arguments.

  • If programs get-edid and parse-edid are installed, command ddcutil environment --verbose uses them for an addiional EDID check

  • assert{} statements are used in libddcutil for situations that should be logically impossible. To aid in remote problem diagnosis, many of these statements have been replaced by macro TRACE_ASSERT(), which additionally writes a messsage to both the libddcutil trace log and the system log.

Preliminary Notes for ddcui Release 0.2.0

This is a major update to ddcui, containing enhanced features, many "fit and finish" changes, and extensive internal rework.

It requires the current version of libddcutil, libddcutil.so.4.1.0, which is built from source package ddcutil 1.2.0. As a distribution installation package, this is normally available as libddcutil4.rpm, libddcutil4.deb, etc.

Configuration file

ddcui uses the configuration file ddcutilrc in two different ways.
- Sections [ddcui] and [global] specify options passed to ddcui. They are prepended to the string entered by the user that is passed to the ddcui command processor.
- Sections [libddcutil] and [global] specify options passed to libddcutil. This is now the preferred method for passing options to the shared library.

For details on the configuration file, see section Config File in the general ddcutil documentation

Previously, several ddcui options were created simply pass a value through to libddcutil. Instead of specifying these options on the ddcui command line, include them in the [libddcutil] section of the configuration file. The use of these ddcui options is deprecated. Some are removed in this version and some or all will be removed in a future release.

Command line options removed in ddcui 0.2.0. Use the configuration file instead.

Removed Option  Purpose
--sleep-multiplier   adjust sleep duration while waiting for a DDC response
--sleep-less, -less-sleep  Do not perform sleep in certain circumstances
--no-sleep-less  Always peform seleep

Options subject to future removal:

Option  Purpose
--ddc   report DDC data errors
--udf, --noudf   enable or disable user defined features
--maxtries   adjust the number of retries for DDC operations
--stats ??   report statistics to the terminal at program termination

--excp |   report internal exceptionss to terminal --trace |   specify a libddcutil trace group --trcfunc |   specify a libddcutil function to be traced --trcfile |   specify a libddcutil file to be traced

Note that the following options are both passed to libddcutil also recognized by the ddcui client. The are not deprecated.

Option  Purpose
--ts, -=timestamp  Prefix trace messages with timestamp
--tid, --threadid  Prefix trace messages with thread id

  • Install ddcui.desktop in /usr/share/applications
  • Install ddcui.appdata.xml in /usr/share/metainfo
  • Install icons in /usr/share/icons/hicolor, /usr/share/pixmaps

Semi-Continuous Values

As of MCCS specification 3.0 and 2.2, several features have both a set of reserved values and with a continuous range. I've dubbed them "semi-continuous". A special widget now exists to represent these features. The widget has both a combo box for the reserved values and a slider/spinbox combination for the continuous range.

The features affected are:

Feature Comment
Feature x62 (Audio Volume)   MCCS 2.2 and 3.0 only
Feature x87 (Audio Treble)   MCCS 2.2 and 3.0 only
X91 (Audio Bass)   MCCS 2.2 and 3.0 only
x93 (Audio Balance)   MCCS 2.2 and 3.0 only

Online Help

Online help is extensively changed

  • All help buttons now provide relevant help
  • The main help screen (accessed from Help->Contents) now links to other help screens.
  • Fomatting of help screens is more consistent
  • Where appropriate, it is possible to navigate backward and forward among screens.

Dialog boxes and help screens are more consistent in style.

  • Rename Options menu entry User Interface Options -> User Interface for cosnsitency
  • Rename dialog Other Options to NC Feature Value

NC Feature Handling

Two mechanisms have been implemented to address the problem of a NC feature values returned by the monitor that is not defined in the MCCS spec or the capabilities string.

  • Sometimes the value is undefined for the monitor's MCCS version, but is defined in a later version. If dialog box option NC Feature Values ->Use Latest NC Values is checked, ddcui will use the value list for the most recent MCCS version that defines the feature, invariably 2.2. Option --use-latest-nc-values can be specified on the command line.

  • If the monitor returns an unrecognzied value, we want to still show that value in the combo box even if another value is selected. This change remembers unrecognized value that have been seen. This memorization does not persist across ddcui calls.

Even with these enhancements, unrecognized NC feature values are probably better handled using a user supplied feature definition. See User Defined Features

NC values handling - if unexpected value occurs, it remains in the drop down menu - Use Latest NC values: if Both or MCCS, uses the latest values if Capabilities, used as text for listed capabilities in both cases, if an unobserved value is detected the value is taken from latest

"use latest NC" option applies to values as well as names

- for MCCS, affects both values and names
- for Capabilites, affects names

Building ddcui

  • File .gitignore has been pruned of entries that do not apply to all git ddcui repos. Locally-specific files to be ignored should be listed in file .git/info/exdclude, which is not under git control.

Miscellaneous

  • The monitor icon previously used by ddcui was replaced by one taken from the Oxygen theme, avoiding any question as to its provenance under an open source license.
  • Option --stats reports execution statistics to the terminal when ddcui terminates normally.
  • Option --version reports the libddcutil and Qt versions that were used for building the programs, and the versions being executed.
  • Key press events are recognized for most, though not all, menu options.
