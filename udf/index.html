User Defined Features

ddcutil* 0.9.9 marks the formal release of User Defined Features. This capability can be enabled or disabled using command line options --udf or --noudef. The default is --udf***.

(This capability has existed for some time but was undocumented as it was subject to change. Until now it was disabled by default.)

The user defined features for a monitor are specified in a feature definition file.

User Defined Feature File

Here's an example of a file that specifies user defined features. 

$ cat DEL-DELL_U3011-16485.mccs
MFG_ID       DEL
MODEL        DELL U3011
PRODUCT_CODE 16485
FEATURE_CODE FD  a Feature  Something
    ATTRS NC RW
    VALUE 0 First Val
    VALUE 0X01 Second Val 0x
    VALUE X02  Third value X
    VALUE 03H  H at end
FEATURE_CODE xFB a name of continuous feature
    ATTRS C RO
FEATURE_CODE 0xfa a feature with code staring with 0x
    ATTRS C ROrock@banner:/shared/playproj/i2c$

The first 3 lines identify the monitor model, using a 3 character manufacturer id, the model name, and the product number, as given in the EDID. The feature definition applies to any monitor that matches those identifiers. Model name and product number are both used because some manufacturer use generic model names, and many set the product number to 0. For example, Samsung commonly uses the name "SyncMaster", and LG commonly uses the name "LG ULTRAWIDE".

These three fields are also are used to create the file name of the feature description, which has the form <mfg_id>-<model name>-<product code>.mccs. The values in the file name must match those in the file.

Following the monitor identification fields are one or more feature descriptions.

Each defnition starts with a line of the form FEATURE_CODE . Feature codes are implicitly hex numbers, and can be written with or without a leading "x". Feature codes must be in the manufacturer reserved range, E0..FF.

The ATTRS line contains mnoitor attributes: "C" or "NC" indicating whether the feature is continuous or non-continuous, and "RW", "RO", or "WO" indicating whether the feature is read-write, read-only, or write-only.

For non-continuous features, there are 1 or more VALUE lines describing the possible values. Each line has the form: "VALUE <value> <description>". Feature values can be written either as decimal or hexadecimal.

The MCCS specification defines non-continuous features that do not have simple tables of feature values. These have special interpretations unique to the each feature. The User Defined Feature facility cannont define such features.

Blank lines and lines beginning with "*" or "#" are ignored.

The format of feature definition files is subject to change in future releases.

Feature Definition Search Path

For now, feature definitions must be found in the user's ddcutil configuration directory, $HOME/.config/ddcutil. Future releases may extend the search path to recognize files in the appropriate directory under /usr/share and the XDG search path.
